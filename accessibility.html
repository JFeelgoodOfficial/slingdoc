Accessibility

Accessibility Statement

Last updated: April 24, 2026

Our commitment

SlingDoc was built for blind and visually impaired workers, field crews, and anyone underserved by conventional workplace software. Accessibility isn't a setting we added — it's the reason the product exists.

Worker experience

Because SlingDoc runs inside the phone's native Messages app, workers use the screen reader, voice assistant, text size, or contrast setting they already have configured — VoiceOver, TalkBack, Voice Control, or otherwise. There is nothing to relearn.

Manager dashboard

Our manager-facing web dashboard targets WCAG 2.2 AA conformance. This includes: proper heading hierarchy, full keyboard operability, visible focus indicators, sufficient color contrast, text alternatives for non-text content, form labels and error messaging, and respect for reduced-motion preferences.

Marketing site

This landing page uses semantic HTML, ARIA live regions to announce animation phases, a skip link for keyboard users, a "reduce motion" toggle, and a static fallback of the hero animation for users who prefer less motion.

Feedback

If you encounter an accessibility barrier, please tell us. Email hello@slingdoc.com. We aim to respond within two business days.
